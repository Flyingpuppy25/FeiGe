FeiGe 0.4.2 更新说明

- 新增 macOS 绿色版构建，支持 Apple 芯片 arm64 与 Intel x64。
- 最低支持 macOS 12；两个架构分别在原生 GitHub macOS 构建机上生成。
- macOS 内置按对应架构编译的 FFmpeg 7.1.5 LGPL 共享运行库。
- macOS 项目数据保存在“~/Library/Application Support/FeiGe”，避免向只读应用包写入数据。
- 增加 macOS 双架构自动构建、SHA-256 校验、应用签名完整性及 FFmpeg 可执行性检查。
- 当前版本没有 Apple Developer 证书，采用临时签名；首次打开需按使用说明确认。

以下为 0.4.1 更新内容：

- 设置页新增一个统一的 API 连接测试按钮，可并行检查全部已完整配置的接口并分别显示结果。
- 项目与风格研究列表统一增加三点菜单：重命名、在文件夹中打开、永久删除；删除前必须确认。
- 移除 Windows 原生 File / Edit / View / Window / Help 菜单栏。
- 风格研究改为四阶段可视流程：准备、分镜拆解、构建拼图、AI 提炼。
- 风格研究会从分镜中均匀选择最多 12 个代表镜头生成拼图，再交给当前模型提炼视觉风格报告。
- 版本号升级为 0.4.1；本次仅生成本地绿色版，不自动上传。

以下为 0.4.0 已有功能：

- 新增四检测器混合拆镜并设为默认：硬切、白闪、渐隐渐现、叠化。
- 保留经典帧差检测模式，新增硬切最低帧差、相对倍率、最大6000镜等设置。
- 新增镜头差异曲线，可单击定位、右键增加切点。
- 分镜支持编辑、删除、重新识别和自动重排时间区间。
- 剧本支持场景级编辑、插入、删除及全文编辑。
- 每次生成剧本创建独立版本，记录 API、模型和时间；支持切换、删除和重新生成。
- OpenAI、Claude与OpenAI兼容接口使用最多24张均匀时间戳关键帧生成剧本。
- Gemini优先使用压缩视频理解，失败时自动回退时间戳关键帧。
- 保留429、5xx、网络错误与超时的四次指数退避重试。
- 新增内容安全策略、自动界面检查、混合算法测试和真实视频媒体测试。
- FFmpeg替换为可再分发的Windows x64 LGPL共享构建，并附完整许可证与来源。
- 继续支持中文 / English 即时切换和1060至2560宽度桌面布局。
- 0.4.0首次启动清空旧项目工作区，保留模型设置和API Key。
